Think Least_Conn In Math

Basic

least_conn is a load balance algorithm, which distributed incoming request to the server with least connected connections.

Think in math

distribute requests to n server, least_conn try to archive a goal every server as busy as others.

total time d that n servers finished requests:

$$\begin{align*} & d = \sum_{i=1}^nd_i = n * d_{avg} \\ \\ & {d_i}\text{ : duration of } i^{th} \text{ server to finish the distributed requests} \\ & {d_{avg}}\text{ : avg duration that a server finished distribted requests} \\ \end{align*}$$

what least_conn algorithm want to archive is :

$$ d_i \approx d_{avg} $$

the duration that a server finished n requests:

$$\begin{align*} d & = \sum_{i=1}^nt_i \\ & = \sum_{i=1}^n(rtt_i + cpu\_real_i + other_i) \\ & = n * t_{avg} \\ & = n * (t_{avg\_net} + t_{avg\_real})\\ & = n * (t_{avg\_net} + (t_{avg\_cpu\_sys\_usr} + t_{avg\_steal}) / cpu\_cores) \\ \\ & {t_{avg\_net}}\text{ : network transfer duration} \\ & {t_{avg\_real}}\text{ : real world time that request finished} \\ & t_{avg\_cpu\_sys\_usr}{\text{ : cpu usage, sys and usr}} \\ & t_{avg\_steal}{\text{ : cpu resource steal by other virtual machine, including cpu throttling time}} \end{align*}$$

Case

1. two same server with different cpu usage show

d: was a server’s time consume to finished the distributed requests, things that already know:

  • two server’s d is equal
  • one server’s cpu usage is lower than the other one
  • the server with lower cpu usage finished least requests at same duration

Think in math:

$$ \begin{align*} \text{ Proof: } \\ &\because \ \exists \ t_{avg\_cpu\_sys\_usr}\downarrow \ \bigwedge \ n \downarrow \\ \\ d &= n\downarrow t_{avg}\uparrow \\ &= n\downarrow *(t_{avg\_net} + ((t_{avg\_cpu\_sys\_usr} + t_{avg\_steal}) / cpu\_cores))\uparrow \\ \\ &\therefore\ t_{avg\_net}\uparrow \ or \ \ (t_{avg\_cpu\_sys\_usr} + t_{avg\_steal})\uparrow \\ &\because \ t_{avg\_cpu\_sys\_usr}\downarrow \\ &\therefore \ t_{avg\_net}\uparrow \ or \ \ t_{avg\_steal}\uparrow \\ Q.E.D \\ \end{align*} $$

Conclusion:

possibilities cause network delay (time of avg_net) increase:

  1. network overload, for example: bandwidth, pps or others
  2. some other network problems

possibilities cause cpu steal increase:

  1. oversolde, when in cloud enviroment

possibilities cause cpu sys_usr decrease:

  1. incorrect system config, show diff with servers, use perf or others generate Flame graphs
  2. cpu throttling, container enviroment which use CFS/cgroup limit cpu usage
  3. oversolde, which cause cpu steal increase
